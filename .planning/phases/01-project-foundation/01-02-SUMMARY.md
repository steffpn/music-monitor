## Decisions Made
- **Composite PK on detections:** TimescaleDB requires the partition column (detected_at) in all unique indexes/PKs. Changed from simple `id` PK to composite `(id, detected_at)`. This is transparent to Prisma Client usage but affects raw SQL queries.
- **FK constraints omitted for hypertable:** Foreign keys referencing or referenced by the detections hypertable are enforced at the Prisma application level, not via SQL constraints. TimescaleDB composite PKs make standard FK references impractical.
- **TS_TUNE_MAX_BG_WORKERS 16:** The latest TimescaleDB pg17 Docker image requires minimum 16 background workers; the plan specified 8 which caused container crash.
- **Weekly aggregate start_offset 28 days:** TimescaleDB requires the refresh window (start_offset - end_offset) to span at least 2 bucket intervals. For 7-day buckets: 28 - 1 = 27 days >= 14 days (2 * 7).
- **AudioSnippet composite FK:** Added `detectionDetectedAt` field to AudioSnippet to reference Detection's composite PK `(id, detectedAt)`.

- **Found during:** Task 1 (migration apply)
- **Issue:** Simple `id` PK on detections table caused `TS103: cannot create a unique index without the column "detected_at"` when converting to hypertable
- **Fix:** Changed to composite PK `(id, detected_at)`, updated unique constraint on `raw_callback_id` to include `detected_at`, updated Prisma schema to use `@@id([id, detectedAt])`
- **Files modified:** migration.sql (initial), schema.prisma
- **Verification:** Migrations apply cleanly, hypertable created successfully
- **Committed in:** 90b3179

**2. [Rule 1 - Bug] TS_TUNE_MAX_BG_WORKERS minimum 16**
- **Found during:** Task 1 (Docker Compose startup)
- **Issue:** TimescaleDB latest-pg17 image requires `maxBGWorkers >= 16`, docker-compose.yml had 8
- **Fix:** Changed `TS_TUNE_MAX_BG_WORKERS` from 8 to 16
- **Files modified:** docker-compose.yml
- **Verification:** Container starts and remains healthy
- **Committed in:** 90b3179

**3. [Rule 1 - Bug] Weekly aggregate refresh policy window too small**
- **Found during:** Task 1 (migration apply, second attempt)
- **Issue:** `start_offset => 14 days` with `end_offset => 1 day` for 7-day bucket: window = 13 days < 14 days (2 * 7 required minimum)
- **Fix:** Changed `start_offset` from `14 days` to `28 days` (covers 4 bucket intervals)
- **Files modified:** timescaledb_setup/migration.sql
- **Verification:** Migration applies successfully, policy created
- **Committed in:** 90b3179

**4. [Rule 1 - Bug] AudioSnippet FK requires composite reference**
- **Found during:** Task 1 (schema consistency)
- **Issue:** AudioSnippet referenced Detection by `id` alone, but Detection now has composite PK `(id, detectedAt)`
- **Fix:** Added `detectionDetectedAt` field to AudioSnippet, updated relation to reference `[id, detectedAt]`, updated SQL migration with composite unique index
- **Files modified:** schema.prisma, migration.sql (initial)
- **Verification:** Prisma generate succeeds, schema is consistent
- **Committed in:** 90b3179
