    **Step 1: Create SSEClient.swift.**
    Create `apps/ios/myFuckingMusic/Services/SSEClient.swift` as an `actor`.

    Properties:
    - `private var task: Task<Void, Never>?` -- the streaming task
    - `private(set) var lastEventId: String?` -- tracks last received event ID for reconnection backfill

    Method `connect(baseURL: URL, token: String) -> AsyncStream<AirplayEvent>`:
    - Build URL: `baseURL.appendingPathComponent("v1/live-feed")` with query item `token=\(token)`
    - Create URLRequest with the URL
    - If lastEventId is set, add header `Last-Event-ID: \(lastEventId)`
    - Return an AsyncStream that:
      - Creates a Task stored in `self.task`
      - Calls `URLSession.shared.bytes(for: request)`
      - Guards response is HTTPURLResponse with statusCode 200
      - Iterates `bytes.lines` (AsyncSequence)
      - Parses SSE wire format line by line:
        - Lines starting with "id:" -> store as currentId
        - Lines starting with "event:" -> store as currentEvent
        - Lines starting with "data:" -> store as currentData
        - Empty line -> end of event frame: if currentEvent == "detection" and currentData is non-nil, decode JSON to AirplayEvent using `JSONDecoder` with `.convertFromSnakeCase` and ISO8601 date strategy, yield to continuation. Update lastEventId = currentId. Reset current* vars.
        - Lines starting with ":" -> comment (keepalive), ignore
      - On error or stream end, call `continuation.finish()`

    NOTE on AirplayEvent decoding from SSE: The SSE data payload is a LiveDetectionEvent JSON (id, stationId, songTitle, artistName, isrc, snippetUrl, stationName, startedAt, publishedAt). This differs slightly from the existing AirplayEvent model (which has endedAt, playCount, createdAt, station.name). Create a private `LiveDetectionEvent` struct inside SSEClient for decoding, then convert to AirplayEvent:
    ```swift
    private struct LiveDetectionEvent: Codable {
        let id: Int
        let stationId: Int
        let songTitle: String
        let artistName: String
        let isrc: String?
        let snippetUrl: String?
        let stationName: String
        let startedAt: String  // ISO 8601
        let publishedAt: String // ISO 8601
    }
    ```
    Convert to AirplayEvent by setting: endedAt = startedAt, playCount = 1, createdAt = startedAt (or publishedAt), station = StationInfo(name: stationName). Use ISO8601DateFormatter for date parsing (reuse the fallback pattern from Phase 6).

    Method `disconnect()`:
    - Cancel `task`, set to nil

    **Step 2: Create LiveFeedViewModel.swift.**
    Create `apps/ios/myFuckingMusic/ViewModels/LiveFeedViewModel.swift`.

    ```swift
    @MainActor
    @Observable
    final class LiveFeedViewModel {
        var events: [AirplayEvent] = []
        var connectionState: ConnectionState = .disconnected
        var newEventCount = 0  // Events arrived while user is scrolled down
        var isAtTop = true     // Track if user is at top of list

        private let sseClient = SSEClient()
        private let maxEvents = 50
        private var disconnectTask: Task<Void, Never>?
        private var connectTask: Task<Void, Never>?

        enum ConnectionState {
            case connected, disconnected, reconnecting
        }
    }
    ```

    Method `connect(token: String)`:
    - Set connectionState = .connected
    - Build baseURL from APIClient's base URL (extract the API base URL -- read it from APIClient or define a shared constant). Use the same base URL pattern as APIClient.
    - Call `await sseClient.connect(baseURL:, token:)` to get AsyncStream
    - Iterate the stream: for each event:
      - If `isAtTop`: insert at index 0 with `withAnimation(.easeInOut(duration: 0.3))`, trim to maxEvents
      - If NOT `isAtTop`: insert at index 0 (no animation to avoid scroll jump), trim to maxEvents, increment newEventCount
    - When stream ends: set connectionState = .disconnected

    Method `reconnect(token: String)`:
    - Set connectionState = .reconnecting
    - Call connect(token:) -- SSEClient will automatically send Last-Event-ID header for backfill

    Method `scrollToTop()`:
    - Reset newEventCount = 0

    Method `scheduleDisconnect(after seconds: Int)`:
    - Create a Task that sleeps for `seconds` then calls `sseClient.disconnect()` and sets connectionState = .disconnected
    - Store in disconnectTask

    Method `cancelScheduledDisconnect()`:
    - Cancel disconnectTask, set to nil

    Method `disconnect()`:
    - Cancel connectTask
    - Call Task { await sseClient.disconnect() }
    - Set connectionState = .disconnected
