    **Step 2: Create SSE route plugin (index.ts).**
    Create `apps/api/src/routes/v1/live-feed/index.ts` as a FastifyPluginAsync. The route:
    - Registers `@fastify/sse` plugin on this scope
    - Defines GET "/" with `{ sse: true }` option (if @fastify/sse supports this pattern; otherwise use reply.sse() approach)
    - Extracts `token` from query params
    - Returns 401 if no token
    - Verifies JWT manually using `fastify.jwt.verify(token)` to get `{ sub: number }` payload (can't use authenticate middleware since auth comes from query param, not Authorization header)
    - Loads user from DB: `prisma.user.findUnique({ where: { id: payload.sub }, include: { scopes: true } })`
    - Returns 401 if user not found or !isActive
    - Builds CurrentUser object from user + scopes (same shape as authenticate middleware)
    - Handles Last-Event-ID backfill:
      - Read `request.headers['last-event-id']`
      - If present, query Redis sorted set: `redis.zrangebyscore(BACKFILL_KEY, parseInt(lastEventId) + 1, "+inf")`
      - Parse each entry as LiveDetectionEvent, filter with shouldDeliverToUser, send as SSE events
    - Creates a dedicated Redis subscriber connection via `createRedisConnection()`
    - Subscribes to `CHANNELS.DETECTION_NEW`
    - On 'message': parse event, call `shouldDeliverToUser(event, currentUser)`, if true send as SSE event with `id: String(event.id)`, `event: "detection"`, `data: JSON.stringify(event)`
    - On connection close (request.raw.on('close')): unsubscribe and disconnect the Redis subscriber
    - Set up keepalive: write SSE comment every 15 seconds (`:keepalive\n\n`) to detect dead connections
