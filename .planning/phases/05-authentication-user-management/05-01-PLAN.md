1. Create `apps/api/src/lib/auth.ts`:
   - `hashPassword(password: string): Promise<string>` -- uses argon2.hash with argon2id, memoryCost 65536, timeCost 3, parallelism 1
   - `verifyPassword(password: string, hash: string): Promise<boolean>` -- uses argon2.verify
   - `generateInviteCode(): string` -- crypto.randomBytes(6).toString("hex").toUpperCase(), format as XXXX-XXXX-XXXX
   - `generateTokenPair(fastify: FastifyInstance, userId: number): Promise<{ accessToken: string; refreshToken: string }>` -- signs JWT access token with { sub: userId } and expiresIn from JWT_ACCESS_EXPIRY constant ("1h"). Generates opaque refresh token via crypto.randomBytes(32).toString("hex"). Creates RefreshToken row in DB with 30-day expiresAt. Returns both tokens.
   - `bootstrapAdmin(): Promise<void>` -- checks prisma.user.count(). If 0, reads ADMIN_EMAIL and ADMIN_PASSWORD from process.env. If both set, hashes password and creates admin user with role "ADMIN", isActive true. Logs result with Pino-compatible console.log.

   - Extend FastifyRequest type declaration to add `currentUser: { id: number; email: string; role: string; isActive: boolean; scopes: Array<{ entityType: string; entityId: number }> }`
   - Export `async function authenticate(request: FastifyRequest, reply: FastifyReply): Promise<void>`:
     - Call `await request.jwtVerify()` in try/catch
     - Extract `sub` from decoded payload (`request.user as { sub: number }`)
     - Load user from DB: `prisma.user.findUnique({ where: { id: sub }, include: { scopes: true } })`
     - If user not found or !user.isActive, return 401
     - Attach `request.currentUser` with id, email, role, isActive, and scopes mapped to `{ entityType, entityId }`
     - On any error, return 401 with `{ error: "Invalid or expired token" }`
