  <behavior>
    Detection worker snippet enqueue (update detection.test.ts):
    - When processCallback creates a NEW AirplayEvent and SNIPPETS_ENABLED='true', it calls snippetQueue.add('extract', { airplayEventId, stationId, detectedAt })
    - When processCallback EXTENDS an existing AirplayEvent, it does NOT enqueue a snippet job
    - When SNIPPETS_ENABLED is not 'true', no snippet job is enqueued even for new AirplayEvents
    - startDetectionWorker accepts an optional snippetQueue parameter for dependency injection

    Snippet API route (airplay-events.test.ts):
    - GET /api/v1/airplay-events/:id/snippet returns { url: string, expiresIn: 86400 } when event has snippetUrl
    - Returns 404 { error: 'Airplay event not found' } when event ID doesn't exist
    - Returns 404 { error: 'No snippet available for this event' } when event.snippetUrl is null
    - Request param :id is validated as integer
  </behavior>
  <action>
    **Modify detection.ts:**
    1. Change `startDetectionWorker` signature to accept an optional `options?: { snippetQueue?: Queue }` parameter
    2. Store the snippet queue reference in module scope (or closure) so `processCallback` can access it
    3. In `processCallback`, after the `await prisma.airplayEvent.create(...)` call (the NEW event branch, when `!recentEvent`):
       - Check `process.env.SNIPPETS_ENABLED === 'true'` AND snippetQueue is set
       - If both true, call `snippetQueue.add('extract', { airplayEventId: newEvent.id, stationId: station.id, detectedAt: detectedAt.toISOString() })`
       - Wrap in try/catch -- log error but don't fail the detection processing (best-effort, non-blocking)
    4. The create call must capture the return value: `const newEvent = await prisma.airplayEvent.create(...)`
    5. Do NOT change the existing AirplayEvent extension branch (the `if (recentEvent)` block)

    **Modify supervisor/index.ts:**
    1. Add import for `startSnippetWorker` from `../../workers/snippet.js`
    2. Start snippet worker BEFORE detection worker: `const { queue: snippetQueue, worker: snippetWorker } = await startSnippetWorker();`
    3. Pass snippet queue to detection worker: `await startDetectionWorker({ snippetQueue });`
    4. Update shutdown sequence (ordered): detectionWorker -> detectionQueue -> snippetWorker -> snippetQueue -> cleanupWorker -> cleanupQueue -> streamManager
       Rationale: detection stops first (no new snippet jobs), then snippet processes remaining jobs, then cleanup runs last

    **Create routes/v1/airplay-events/schema.ts:**
    - `AirplayEventParams`: Type.Object with `id: Type.Number()` for path params
    - `SnippetUrlResponse`: Type.Object with `url: Type.String()` and `expiresIn: Type.Number()`
    - `ErrorResponse`: Type.Object with `error: Type.String()` (reuse pattern from stations)
    - Export static types: `AirplayEventParams`

    **Create routes/v1/airplay-events/handlers.ts:**
    - `getSnippetUrl(request, reply)`:
      1. Parse `request.params.id` as integer
      2. `prisma.airplayEvent.findUnique({ where: { id } })`
      3. If not found, reply 404 `{ error: 'Airplay event not found' }`
      4. If `snippetUrl` is null, reply 404 `{ error: 'No snippet available for this event' }`
      5. Call `getPresignedUrl(event.snippetUrl, 86400)` from lib/r2.js
      6. Reply 200 `{ url: presignedUrl, expiresIn: 86400 }`
    - NOTE: Auth middleware is Phase 5. For now, endpoint is unauthenticated. Add a comment: `// TODO: Phase 5 -- add JWT auth middleware`

    **Create routes/v1/airplay-events/index.ts:**
    - Fastify plugin registering GET `/:id/snippet` with TypeBox schema validation on params and response
    - Follow exact pattern from stations/index.ts

    **Update routes/v1/index.ts:**
    - Add: `fastify.register(import("./airplay-events/index.js"), { prefix: "/airplay-events" });`

    **Update detection.test.ts:**
    - Add a mock snippet queue (`{ add: vi.fn().mockResolvedValue(undefined) }`)
    - Set `process.env.SNIPPETS_ENABLED = 'true'` in relevant tests
    - Test: new AirplayEvent creation calls snippetQueue.add with correct job data
    - Test: extending existing AirplayEvent does NOT call snippetQueue.add
    - Test: SNIPPETS_ENABLED='false' prevents snippetQueue.add even for new events
    - Ensure existing detection tests still pass (snippet queue is optional)

    - Mock getPresignedUrl from lib/r2.js
    - Test 200 response with presigned URL when snippetUrl exists
    - Test 404 when event not found
    - Test 404 when snippetUrl is null
