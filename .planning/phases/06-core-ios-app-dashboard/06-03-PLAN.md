    **DashboardModels.swift:** Create Codable, Sendable response types:
    - `enum TimePeriod: String, CaseIterable { case day, week, month }` with display label computed property ("Today", "This Week", "This Month") and a calendarComponent (Calendar.Component) for chart axis formatting
    - `struct DashboardSummaryResponse: Codable, Sendable { let buckets: [PlayCountBucket]; let totals: PlayCountTotals }`
    - `struct PlayCountBucket: Codable, Identifiable, Sendable { let bucket: String; let playCount: Int; let uniqueSongs: Int; let uniqueArtists: Int; var id: String { bucket } }` -- Note: bucket comes as ISO date string from backend
    - `struct PlayCountTotals: Codable, Sendable { let playCount: Int; let uniqueSongs: Int; let uniqueArtists: Int }`
    - `struct TopStationsResponse: Codable, Sendable { let stations: [StationPlayCount] }`
    - `struct StationPlayCount: Codable, Identifiable, Sendable { let stationId: Int; let stationName: String; let playCount: Int; var id: Int { stationId } }`

    **DashboardViewModel.swift:** @Observable @MainActor class:
    - Properties: selectedPeriod (TimePeriod = .day), summaryResponse (DashboardSummaryResponse?), topStations ([StationPlayCount] = []), isLoading (Bool = false), error (String?)
    - loadDashboard(): Sets isLoading, calls both dashboardSummary and topStations endpoints via APIClient in parallel (async let), stores results, clears loading. On error, sets error string.
    - Computed properties for SummaryCardsView: todayPlays, weekPlays, monthPlays derived from totals.
    - When selectedPeriod changes, re-fetch data. Use Swift's .onChange(of:) or .task(id:) in the view to trigger.

    **DashboardView.swift:** Main dashboard tab view:
    - @State private var viewModel = DashboardViewModel()
    - Picker with SegmentedPickerStyle for Day | Week | Month at the top
    - ScrollView containing vertically stacked sections:
      1. SummaryCardsView(totals: viewModel.summaryResponse?.totals)
      2. PlayCountChartView(data: viewModel.summaryResponse?.buckets ?? [], period: viewModel.selectedPeriod)
      3. TopStationsView(stations: viewModel.topStations)
    - .task(id: viewModel.selectedPeriod) { await viewModel.loadDashboard() } -- auto-reloads on period switch
    - Show LoadingView() when isLoading and no data yet
    - Show ErrorView(message:retryAction:) when error is set
    - Use .refreshable { await viewModel.loadDashboard() } for pull-to-refresh

    **SummaryCardsView.swift:** Horizontal row of 3 cards:
    - Each card: VStack with bold large number (.title font) and label ("Today", "This Week", "This Month")
    - Use HStack with equal spacing or LazyHGrid
    - Cards styled with RoundedRectangle background, shadow, padding
    - Show "--" placeholder when totals is nil (loading state)
    - Also show uniqueSongs and uniqueArtists as smaller text below each count

    **PlayCountChartView.swift:** Using Swift Charts framework:
    - Import Charts
    - Parse bucket string to Date using ISO8601DateFormatter
    - BarMark(x: .value("Date", bucketDate), y: .value("Plays", playCount)) for each bucket
    - .foregroundStyle(.blue.gradient)
    - Chart X-axis formatted by period (day: hour, week: day abbreviation, month: day number)
    - .frame(height: 200)
    - Show "No data" text when data array is empty
    - Section header "Play Count Trend"

    **TopStationsView.swift:** Using Swift Charts horizontal bar chart:

    - .frame(height: CGFloat(stations.count * 44)) -- dynamic height
