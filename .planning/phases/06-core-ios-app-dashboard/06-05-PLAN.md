    **AudioPlayerManager.swift:** @Observable @MainActor class for snippet playback:
    - Properties:
      - currentlyPlayingId: Int? = nil (which event is playing)
      - playbackProgress: Double = 0 (0.0 to 1.0)
      - isPlaying: Bool = false
      - isLoadingSnippet: Bool = false (while fetching presigned URL)
      - error: String? = nil
    - Private properties:
      - player: AVPlayer? = nil
      - timeObserver: Any? = nil

    - play(eventId: Int): async method that:
      1. If currentlyPlayingId == eventId and isPlaying, call pause(). Return.
      2. If currentlyPlayingId == eventId and !isPlaying, call resume(). Return.
      3. stop() -- stop any current playback
      4. Set isLoadingSnippet = true, currentlyPlayingId = eventId
      5. Fetch fresh presigned URL via APIClient: let response: SnippetUrlResponse = try await APIClient.shared.request(.snippetUrl(eventId: eventId))
      6. Set isLoadingSnippet = false
      7. Configure AVAudioSession: try AVAudioSession.sharedInstance().setCategory(.playback) -- IMPORTANT per research pitfall 3
      8. Create AVPlayerItem(url: URL(string: response.url)!)
      9. Create AVPlayer(playerItem: item)
      10. Add periodic time observer (every 0.1s) updating playbackProgress = currentTime / duration
      11. Add NotificationCenter observer for .AVPlayerItemDidPlayToEndTime to call stop()
      12. player?.play(), set isPlaying = true
      13. On any error: set error message, call stop()

    - pause(): player?.pause(), isPlaying = false
    - resume(): player?.play(), isPlaying = true
    - stop(): Remove time observer, remove notification observer, player?.pause(), player = nil, currentlyPlayingId = nil, isPlaying = false, playbackProgress = 0

    IMPORTANT anti-patterns to avoid (from research):
    - Always stop() before starting new playback (prevent multiple simultaneous streams)
    - Set AVAudioSession category before first play
    - Time observer uses [weak self] or captures reference safely
    - Handle CancellationError from Task if view disappears during URL fetch

    **SnippetPlayerView.swift:** Inline expanded player shown below the detection row:
    - Properties: playbackProgress (Double), isPlaying (Bool), onPlayPause (closure), onStop (closure)
    - Layout: VStack with:
      - Thin ProgressView(value: playbackProgress) (.tint(.blue), .scaleEffect(y: 0.5)) as horizontal progress bar
      - HStack with:
        - Play/Pause button: Image(systemName: isPlaying ? "pause.circle.fill" : "play.circle.fill") sized 24x24
        - Text showing time: formatted as "0:02 / 0:05" (current / total)
        - Spacer
        - Stop button: Image(systemName: "xmark.circle") to stop and collapse
    - Compact height -- the entire player view should be about 40 points tall
    - Animated appearance with .transition(.opacity.combined(with: .move(edge: .top)))
