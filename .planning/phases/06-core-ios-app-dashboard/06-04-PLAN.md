    **PaginatedResponse.swift:** Generic paginated response wrapper:
    ```swift
    struct PaginatedResponse<T: Codable & Sendable>: Codable, Sendable {
        let data: [T]
        let nextCursor: Int?
    }
    ```

    **Detection.swift modification:** Add a nested station struct to AirplayEvent for the included station name:
    - Add `struct StationInfo: Codable, Sendable { let name: String }` inside or alongside AirplayEvent
    - Add `let station: StationInfo?` property to AirplayEvent (optional since existing uses may not include it)
    - Keep existing Detection struct unchanged

    **DateFormatters.swift:** Shared date formatting utilities:
    - `static let iso8601: ISO8601DateFormatter` -- for parsing API dates (backup for when JSONDecoder strategy doesn't cover all formats)
    - `static func relativeTime(_ date: Date) -> String` -- "2h ago", "Yesterday", "Mar 15" style relative formatting using RelativeDateTimeFormatter for recent dates and DateFormatter for older
    - `static func shortDateTime(_ date: Date) -> String` -- "Mar 15, 2:30 PM" format for detection row timestamps
    - `static func dateOnly(_ date: Date) -> String` -- "Mar 15, 2026" for filter display
    - `static func isoDateString(_ date: Date) -> String` -- "2026-03-15" format for API query parameters

    **DetectionsViewModel.swift:** @Observable @MainActor class:
    - Properties:
      - detections: [AirplayEvent] = []
      - isLoading: Bool = false
      - isLoadingMore: Bool = false
      - nextCursor: Int? = nil
      - hasMore: Bool { nextCursor != nil }
      - searchQuery: String = ""
      - startDate: Date? = nil
      - endDate: Date? = nil
      - selectedStationId: Int? = nil
      - stations: [Station] = [] (for filter picker)
      - error: String? = nil
    - loadInitial(): Clears detections, resets cursor, fetches first page. Called on view appear and on filter/search change.
    - loadMore(): Guard with !isLoadingMore && hasMore. Fetches next page using cursor. Appends to detections array.
    - refresh(): Same as loadInitial but called from pull-to-refresh.
    - loadStations(): Fetch station list for filter picker. Cache in property. Called once on view appear.
    - Private fetchPage(cursor:) -> PaginatedResponse<AirplayEvent>: Calls APIClient with .airplayEvents endpoint, passing current searchQuery, date range (formatted as ISO strings), stationId, cursor, limit: 20.

    Search debouncing: Use .task(id: searchQuery) in the view with a 300ms delay via `try await Task.sleep(for: .milliseconds(300))` before calling loadInitial(). SwiftUI .task(id:) auto-cancels previous tasks when searchQuery changes, providing natural debounce.

    Filter changes: When startDate, endDate, or selectedStationId change, call loadInitial() to reset and refetch with new filters.

    **Xcode project:** Add new files.
