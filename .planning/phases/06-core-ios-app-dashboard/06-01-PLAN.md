<task type="auto" tdd="true">
  <name>Task 1: Dashboard aggregate API endpoints (GET /dashboard/summary and /dashboard/top-stations)</name>
  <files>
    apps/api/src/routes/v1/dashboard/index.ts,
    apps/api/src/routes/v1/dashboard/schema.ts,
    apps/api/src/routes/v1/dashboard/handlers.ts,
    apps/api/src/routes/v1/index.ts,
    apps/api/tests/routes/dashboard.test.ts
  </files>
  <behavior>
    - GET /dashboard/summary?period=day returns { buckets: [{ bucket, playCount, uniqueSongs, uniqueArtists }], totals: { playCount, uniqueSongs, uniqueArtists } } from daily_station_plays for last 1 day
    - GET /dashboard/summary?period=week returns data from daily_station_plays for last 7 days
    - GET /dashboard/summary?period=month returns data from daily_station_plays for last 30 days
    - GET /dashboard/summary without period defaults to "day"
    - GET /dashboard/top-stations?period=day&limit=10 returns { stations: [{ stationId, stationName, playCount }] } sorted by playCount desc
    - GET /dashboard/top-stations without limit defaults to 10
    - Both endpoints require authentication (401 without JWT)
    - STATION role users see only data for their scoped station IDs
    - ADMIN role users see all data
    - ARTIST/LABEL roles see all data (scope filtering deferred per Phase 5 decision)
  </behavior>
  <action>
    Follow the exact Fastify plugin route pattern used by all prior phases: routes/v1/dashboard/index.ts + schema.ts + handlers.ts.

    **schema.ts:** Create TypeBox schemas:
    - DashboardSummaryQuerySchema: { period: Type.Optional(Type.Union([Type.Literal("day"), Type.Literal("week"), Type.Literal("month")])) }
    - TopStationsQuerySchema: { period: Type.Optional(Type.Union([...])), limit: Type.Optional(Type.Number({ minimum: 1, maximum: 50, default: 10 })) }
    - Response schemas for type documentation.

    **handlers.ts:** Two handlers:
    - `getDashboardSummary`: Uses prisma.$queryRaw to query daily_station_plays continuous aggregate. Build scope filter for STATION role. Period maps to interval: day=1 day, week=7 days, month=30 days. Return both bucketed data and totals (SUM across all buckets). Cast BigInt results to Number.
    - `getTopStations`: Uses prisma.$queryRaw on daily_station_plays joined with stations table for station name. GROUP BY station_id, ORDER BY play_count DESC, LIMIT from query. Apply same scope filter.

    IMPORTANT: Prisma $queryRaw returns BigInt for COUNT aggregates in PostgreSQL. Cast with ::int in SQL or convert in JS with Number(). Use Prisma.sql for parameterized queries (not string interpolation) to prevent SQL injection. Use Prisma.join() for IN clauses with station ID arrays.

    **index.ts:** Register both GET routes with [authenticate] preHandler and schema validation.

    **v1/index.ts:** Add dashboard route registration: `fastify.register(import("./dashboard/index.js"), { prefix: "/dashboard" });`

    **Test file (dashboard.test.ts):** Follow the exact same pattern as airplay-events.test.ts:
    - Mock prisma with vi.mock, including $queryRaw mock
    - Mock r2, redis as stubs
    - Create server instance, call server.ready()
    - Use the test helper's getAuthTokens pattern to inject JWT
    - Test each endpoint with valid/invalid params, auth checks, scope filtering
  </action>
  <verify>
    <automated>cd apps/api && pnpm test -- --run tests/routes/dashboard.test.ts</automated>
  </verify>
  <done>Dashboard summary endpoint returns bucketed play counts and totals for day/week/month. Top stations endpoint returns ranked station list. Both require auth and filter by scope. All tests pass.</done>
</task>

    - GET /airplay-events?cursor=50&limit=10 returns events with id less than 50, limited to 10
    - GET /airplay-events?q=Smiley searches songTitle, artistName (case-insensitive contains), and isrc (case-insensitive equals)
    - GET /airplay-events?startDate=2026-03-01&endDate=2026-03-15 filters by startedAt date range
    - GET /airplay-events?stationId=5 filters by station
    - Results include station name via include (station: { select: { name: true } })
    - Results ordered by startedAt DESC (newest first)
    - nextCursor is null when no more results
    - Requires authentication (401 without JWT)
    - STATION role only sees events from scoped stations
    - ADMIN sees all events
    - ARTIST/LABEL see all events (scope filtering deferred)
    - Existing GET /:id/snippet route remains unchanged
  </behavior>
  <action>
    Extend the existing airplay-events route group. Do NOT modify the existing getSnippetUrl handler.

    **schema.ts additions:**
    - ListEventsQuerySchema: Type.Object with cursor (Optional Number), limit (Optional Number, default 20, max 100), q (Optional String), startDate (Optional String format date), endDate (Optional String format date), stationId (Optional Number)
    - ListEventsResponseSchema: Type.Object({ data: Type.Array(AirplayEventSchema), nextCursor: Type.Union([Type.Number(), Type.Null()]) })
    - AirplayEventSchema: Type.Object with all AirplayEvent fields plus station: Type.Object({ name: Type.String() })

    **handlers.ts addition -- listEvents function:**
    - Build Prisma where clause from query params
    - Scope filter: STATION role restricts to scoped stationIds (same pattern as getSnippetUrl)
    - Search (q param): Use OR with contains/insensitive on songTitle, artistName, and equals/insensitive on isrc
    - Date range: startedAt >= startDate AND startedAt <= endDate
    - Station filter: stationId equals
    - Cursor: id < cursor (descending order pagination)
    - Fetch limit + 1 to determine hasMore. Slice to limit. nextCursor = last item id if hasMore.
    - Include station name via include: { station: { select: { name: true } } }
