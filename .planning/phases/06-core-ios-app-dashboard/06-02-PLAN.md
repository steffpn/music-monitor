<interfaces>
<!-- Backend auth API contracts the iOS app must call -->

POST /api/v1/auth/register
  Body: { code: string (14 chars), email: string, password: string (min 8), name: string }
  Response 201: { user: { id, email, name, role }, accessToken: string, refreshToken: string }
  Response 400: { error: string } (invalid code)
  Response 409: { error: string } (email exists)

POST /api/v1/auth/login
  Body: { email: string, password: string }
  Response 200: { user: { id, email, name, role }, accessToken: string, refreshToken: string }
  Response 401: { error: "Invalid credentials" }

POST /api/v1/auth/refresh
  Body: { refreshToken: string }
  Response 200: { accessToken: string, refreshToken: string }
  Response 401: { error: "Invalid refresh token" }

POST /api/v1/auth/logout (requires Bearer token)
  Body: { refreshToken: string }
  Response 200: { message: "Logged out" }

  <action>
    **KeychainHelper.swift:** Create a struct with static methods wrapping Security.framework. Use service identifier "com.myfuckingmusic.auth". Methods: save(key:value:), read(key:) -> String?, delete(key:), clearAll(). Follow the exact pattern from RESEARCH.md code example. Keychain ops are fast synchronous C calls -- nonisolated static methods are appropriate.

    **AuthResponse.swift:** Create response model structs:
    - AuthResponse: Codable, Sendable -- { user: AuthUser, accessToken: String, refreshToken: String }
    - AuthUser: Codable, Sendable -- { id: Int, email: String, name: String, role: String }
    - TokenResponse: Codable, Sendable -- { accessToken: String, refreshToken: String } (for refresh endpoint)
    - ErrorResponse: Codable, Sendable -- { error: String }

    **AuthManager.swift:** Create an @Observable class (NOT actor -- needs to be MainActor-accessible for UI binding):
    - Properties: isAuthenticated (Bool), currentUser (AuthUser?), isLoading (Bool)
    - checkStoredTokens(): Read tokens from Keychain. If both exist, try to call APIClient to validate (attempt refresh). Set isAuthenticated accordingly. If refresh fails, clearTokens and set isAuthenticated = false.
    - register(code:email:password:name:): Call register endpoint via APIClient. On success, store tokens in Keychain, set currentUser, set isAuthenticated = true.
    - login(email:password:): Call login endpoint. On success, store tokens, set currentUser, set isAuthenticated.
    - logout(): Call logout endpoint (best-effort, don't fail on network error). Clear Keychain. Set isAuthenticated = false, currentUser = nil.
    - refreshAccessToken(): Actor-based single-flight pattern. Uses a private refreshTask: Task<String, Error>? to coalesce concurrent refresh calls. Reads refresh token from Keychain, calls POST /auth/refresh, stores new tokens in Keychain. If refresh fails, clears tokens and sets isAuthenticated = false (silent redirect to login).
    - Mark class as @MainActor for UI-safe property updates.

    **APIClient.swift:** Modify the existing actor:
    - Add a property `var authManager: AuthManager?` set via a configure method (avoids circular init).
    - Update the `request<T>` method to:
      1. Read access token from KeychainHelper.read(key: "accessToken") and add "Authorization: Bearer {token}" header if present
      2. On 401 response: attempt token refresh via authManager?.refreshAccessToken(), then retry the original request ONCE with the new token
      3. If retry also fails with 401, throw the error (AuthManager will handle logout)
    - Add an `encoder` property (JSONEncoder with .convertToSnakeCase) for POST request bodies.
    - Add a method `requestRaw(_ endpoint: APIEndpoint) async throws -> (Data, HTTPURLResponse)` for cases where callers need raw response handling.

    **APIEndpoint.swift:** Replace the stub with complete endpoint enum covering ALL Phase 6 needs:
    - .health
    - .register(code:email:password:name:), .login(email:password:), .refresh(refreshToken:), .logout(refreshToken:)
    - .dashboardSummary(period:), .topStations(period:limit:)
    - .airplayEvents(cursor:limit:query:startDate:endDate:stationId:), .snippetUrl(eventId:)
    - .stations
    - Computed properties: path (String), method (HTTPMethod), body (Data?), queryItems ([URLQueryItem]?)
    - For GET endpoints with query params, return queryItems. For POST endpoints, return JSON body.
    - Update APIClient.request to build URL with queryItems when present.

    **Xcode project:** Add all new .swift files to the project.pbxproj. Each file needs a PBXBuildFile entry, a PBXFileReference entry, and inclusion in the appropriate PBXGroup. Follow the existing file reference pattern. Alternatively, if the project uses folder references, ensure files are in the correct directories.

    IMPORTANT: Use @Observable macro (NOT ObservableObject/@Published). Use .environment() (NOT @EnvironmentObject) for passing through SwiftUI. iOS 17 minimum target.
